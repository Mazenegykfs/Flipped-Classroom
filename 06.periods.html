Using Periods in Technical Writing

1. Objectives

  • Correctly use a period to end declarative and mild imperative sentences.
  • Identify and correct run-on sentences and sentence fragments.
  • Use periods correctly in common abbreviations (e.g., etc., i.e.).
  • Understand that a period conveys a neutral, factual tone suitable for professional writing.

2. Review

A period (.), also known as a full stop, is a punctuation mark used to signal the end of a complete declarative sentence (a statement) or a mild imperative sentence (a command).

3. Warm-up

How does punctuation create clarity in these examples?

1. The system rebooted the update was successful the server is now online.

2. The system rebooted. The update was successful. The server is now online.

Sentence 1 is a run-on sentence, making it confusing and hard to read. Sentence 2 uses periods to separate the three distinct ideas into clear, professional statements, which improves readability.

4. Presentation: Two Key Period Rules

Rule 1: To End Sentences

Use a period to mark the end of a complete thought that is a statement or a mild command. This is the most common use of the period.

Declarative: The new algorithm improved efficiency by 20 percent.

Imperative: Please submit the report by 5 PM.

Rule 2: To Mark Abbreviations

Periods are used in many common abbreviations. Note that in modern technical writing, some abbreviations (especially acronyms) are often written without periods (e.g., NASA, CEO).

Example: The presentation is scheduled for 9 a.m.

Example: Please include all relevant materials, e.g., schematics, data sheets, etc.
